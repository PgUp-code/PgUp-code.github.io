1. 文档基本信息​

属性说明
id文档的唯一标识，默认为文件名（不带扩展名）。
title文档标题，默认使用 # 一级标题 的内容。
slug自定义 URL 路径，默认使用 id。
description文档的简要描述，影响 SEO。
示例​
---
id: intro
title: 介绍 Docusaurus
slug: /custom-url
description: 这是 Docusaurus 文档的介绍页面，帮助用户快速了解。
---

🔹 备注：

  • slug 用于指定该页面的 URL，默认路径是 /docs/intro，如果改成 /custom-url，访问路径就是 /custom-url。
  • description 可用于搜索引擎优化（SEO）。

2. 侧边栏相关​

属性说明
sidebar_label在侧边栏显示的名称，默认使用 title。
sidebar_position侧边栏排序，数值越小越靠前。
sidebar_class_name侧边栏 CSS 类，可用于自定义样式。
sidebar_custom_props自定义属性，可用于组件扩展。
示例​
---
title: 教程
sidebar_label: 🚀 快速入门
sidebar_position: 1
sidebar_class_name: highlight
sidebar_custom_props:
  tooltip: "点击这里查看教程"
---

🔹 备注：

  • sidebar_label 会替换默认的 title 在侧边栏显示名称。
  • sidebar_class_name 允许添加自定义 CSS，例如 .highlight { color: red; }。
  • sidebar_custom_props 可用于传递额外属性，例如 tooltip。

3. SEO 相关​

属性说明
keywords关键字列表，有助于 SEO。
image指定 SEO 和社交媒体分享的封面图片。
示例​
---
title: Docusaurus SEO
keywords: [Docusaurus, SEO, 文档]
description: 了解如何优化 Docusaurus 站点的 SEO。
image: /img/docusaurus-social.png
---

🔹 备注：

  • keywords 用于提升搜索引擎索引质量。
  • image 主要用于社交分享时的预览图，例如在 Twitter 或 Facebook 上分享链接时。

4. 标签（Tags）​

属性说明
tags文档的标签，可以用于分类和搜索。
示例​
---
title: Docusaurus 教程
tags: [教程, 快速入门, 开发]
---

🔹 备注：

  • tags 可用于筛选和组织文档，例如在博客页面中使用。

5. 博客专用 Front Matter​

如果启用了 @docusaurus/plugin-content-blog，可以使用以下 Front Matter：

属性说明
authors作者信息，可以是字符串或数组。
date文章发布日期，格式 YYYY-MM-DD。
draft是否为草稿文章，true 则不会发布。
hide_table_of_contents是否隐藏文章目录（TOC）。
示例​
---
title: Docusaurus 博客
authors: ["张三", "李四"]
date: 2024-03-10
draft: false
hide_table_of_contents: false
---

🔹 备注：

  • authors 可用于显示作者信息。
  • draft: true 时，该文章不会显示在博客列表中。

6. 重定向​

属性说明
redirect_from允许将旧 URL 重定向到新 URL。
示例​
---
title: 新教程
redirect_from:
  - /old-path
  - /docs/old-tutorial
---

🔹 备注：

  • 访问 /old-path 或 /docs/old-tutorial 会自动跳转到该页面。

7. 自定义 Front Matter​

你还可以定义自己的 Front Matter，Docusaurus 允许在插件或组件中读取它们。例如：

---
title: 自定义属性示例
customProps:
  difficulty: "高级"
  estimated_reading_time: "10 分钟"
---

🔹 备注：

  • customProps 可用于自定义组件读取数据，例如文档难度、预计阅读时间等。

文件夹设置​

在相应文件夹下，新增_category_.json文件，里面记录了排序位置，文件夹说明等设置。具体内容如下：

{
  "label": "Docusaurus 日常使用", // 文件夹名称
  "position": 1, // 文件夹排序
  "link": {
    "type": "generated-index",
    "description": "Docusaurus 增删改查" // 文件夹说明
  }
}
